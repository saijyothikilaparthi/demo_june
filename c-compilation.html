Pipeline Order

.c → .i → .ll/.bc → .s → .o → Executable

Compiler Stages

Preprocessor → Clang Front-End → LLVM Back-End → Assembler → Linker

Generate All Files

clang --save-temps -c file.c

Main Idea

Clang always passes through LLVM IR before generating Assembly Code.

📘 One-Page Command Reference

Goal Command
Preprocessed Source (.i) clang -E file.c -o file.i
Without Line Markers clang -E -P file.c
All Intermediate Files clang --save-temps -c file.c
Generate Text IR (.ll) clang -emit-llvm -S file.c -o file.ll
Generate Binary IR (.bc) clang -emit-llvm -c file.c -o file.bc
Bitcode → Text IR llvm-dis file.bc -o file.ll
Native Assembly (.s) clang -S file.c
Compare Optimizer clang -O0/-O2 -emit-llvm -S file.c
Remember the Order

.c → .i → IR (.ll/.bc) → .s → .o → Executable
