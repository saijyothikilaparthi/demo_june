🐞 Debugging using LLDB

🐞 What is Debugging?

Debugging is the process of finding and fixing errors in a program. Instead of guessing, a debugger runs the program step by step and shows the values of variables.

🔄 Debug Loop

Write Code
⬇

⬇
Start LLDB
⬇
Breakpoint
⬇

⬇
Next → Check Variables
⬇
Quit

⚙ Compile with Debug Info

clang -g file.c -o file
-g adds debugging information so LLDB can map machine code back to C source code.

🚀 Start LLDB

lldb ./file

Loads the executable into the debugger.

🛑 Common LLDB Commands

Command Purpose
b main Breakpoint at main()
run Start Program
next Execute Next Line
frame variable Show Variables
print total Print One Variable
continue Run Until Next Stop
quit Exit LLDB

💻 Simple Debug Example

clang -g main.c -o main

lldb ./main

b main

run

next

frame variable

continue

quit

Watch the program line by line and observe variable values.
