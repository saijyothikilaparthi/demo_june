Environment Variables

A simple way for the Operating System to store important information that programs can read and use while running.

What are Environment Variables?

Environment Variables are Name = Value pairs.

📦 Name = Value

Every Environment Variable has a unique name (Key) and stores a value. Programs access these values whenever they need information.

JAVA_HOME

📚 Think of it as Labeled Boxes

Imagine your computer keeps many labeled storage boxes. Each box has a label (Variable Name) and stores useful information (Value). Whenever a program needs information, it simply opens the correct box.

🛠 Temporary & Permanent Variables

Temporary variables exist only for the current terminal session. Permanent variables remain available even after restarting the terminal.

⏳ Temporary Variables

Create a variable for the current terminal session.


export my_var=hello

echo $my_var

unset my_var

⭐ Remember

  • Environment Variable = Name + Value
  • Programs read values at runtime.
  • PATH stores executable locations.
  • Temporary variables disappear after closing the terminal.
  • Permanent variables remain after restart.
  • Use env, printenv or set to list variables.
